./db/ - дамп базы
./config/config.php - данные для доступа к MySQL

Старался эксперементировать, создал 2 фабрики объектов, которые могут создавать объекты 3 разных классов Renderer_...  
в зависимости от переданного в конструктор фабрики имени класса рендера. 
Клаcc Сontroller может одинаково работать со всеми фабриками и соответсвенно со всеми рендерами.

Над интерфейсом для тестирования сильно не замарачивался, сделал несколько готовых URL. 

Выбор фабрики и рендера определяется через url:
Фабрика - factory может быть равным RenderFactoryA или RenderFactoryB

Рендерер - renderer может принимать значения default, alpha, betta.
Различия между ними совсем небольшие, но этого уже второстепенные художества.

Архитектура MVC и принципы SOLID в общем-то потихоньку укладываются в голове, но еще сложно быстро сориентироваться, представить
полностью всю картину (структуру программы) в голове.
